# Go Application with GORM and PostgreSQL

## Описание проекта

Это Go-приложение с использованием GORM ORM для работы с PostgreSQL. Проект включает модели для системы с пользователями, чатами, сообщениями, токенами и связями между пользователями.

## Структура проекта

```
.
├── main.go                 # Точка входа в приложение
├── go.mod                  # Файл модуля Go с зависимостями
├── .env.example            # Пример конфигурации окружения
├── models/
│   └── models.go          # Определения моделей базы данных
└── database/
    ├── database.go        # Подключение к базе данных
    └── migrate.go         # Миграции базы данных
```

## Модели данных

### User (Пользователь)
- Основная сущность системы
- Содержит личную информацию: имя, email, логин, страна, город, дата рождения, пол
- Уникальный логин для каждого пользователя
- Связи с токенами, чатами и другими пользователями

### LinkToken (Токен)
- Токены для авторизации или специальных ссылок
- Связан с пользователем
- Имеет время создания и истечения

### Chat (Чат)
- Чаты, созданные пользователями
- Статус активности (is_active)
- Связь с пользователем-создателем и сообщениями

### Message (Сообщение)
- Сообщения в чатах
- Текст сообщения и время создания
- Связь с чатом

### Relation (Связь)
- Связи между двумя пользователями
- Может представлять дружбу, подписку и т.д.

## Установка и настройка

### 1. Предварительные требования

- Go 1.21 или выше
- PostgreSQL 12 или выше
- Git

### 2. Клонирование репозитория

```bash
git clone <your-repo-url>
cd backend
```

### 3. Установка зависимостей

```bash
go mod download
```

### 4. Настройка базы данных

#### Создание базы данных PostgreSQL:

```sql
CREATE DATABASE myapp_db;
```

#### Настройка переменных окружения:

```bash
# Скопируйте пример конфигурации
cp .env.example .env

# Отредактируйте .env файл с вашими параметрами
nano .env
```

Или экспортируйте переменные окружения напрямую:

```bash
export DB_HOST=localhost
export DB_PORT=5432
export DB_USER=postgres
export DB_PASSWORD=your_password
export DB_NAME=myapp_db
export DB_SSLMODE=disable
```

### 5. Запуск приложения

```bash
go run main.go
```

При первом запуске приложение автоматически:
- Подключится к PostgreSQL
- Создаст все необходимые таблицы
- Применит индексы и ограничения
- Создаст тестовые данные (пользователя, чат, сообщение)

## Использование

### Пример работы с моделями:

```go
// Создание нового пользователя
user := &models.User{
    Name:     "John Doe",
    Email:    "john@example.com",
    Login:    "johndoe",
    Password: "hashed_password",
}
database.DB.Create(user)

// Поиск пользователя
var foundUser models.User
database.DB.Where("login = ?", "johndoe").First(&foundUser)

// Создание чата для пользователя
chat := &models.Chat{
    UserID:   foundUser.UserID,
    ChatName: "My Chat",
}
database.DB.Create(chat)

// Загрузка пользователя с связанными данными
database.DB.Preload("Chats").Preload("LinkTokens").First(&foundUser, foundUser.UserID)
```

## Безопасность

⚠️ **Важно**: В примере используются простые пароли для демонстрации. В реальном приложении:
- Всегда хешируйте пароли (используйте bcrypt или argon2)
- Используйте подготовленные запросы (GORM делает это автоматически)
- Валидируйте входные данные
- Используйте HTTPS для API
- Настройте правильные права доступа к БД

## Миграции

Приложение использует автоматические миграции GORM, которые:
- Создают новые таблицы
- Добавляют новые колонки
- Создают индексы
- НЕ удаляют существующие колонки (для безопасности данных)

Для полного пересоздания схемы БД можно использовать:
```go
database.DropTables(database.DB)  // Удалит все таблицы
database.Migrate(database.DB)     // Создаст заново
```

## Troubleshooting

### Ошибка подключения к БД
- Проверьте, что PostgreSQL запущен
- Проверьте правильность параметров подключения
- Убедитесь, что база данных создана

### Ошибка миграции
- Проверьте права пользователя БД на создание таблиц
- Посмотрите логи для деталей ошибки
